The first public chronicle for Shore Assistant. This entry covers the work
done from the initial commit through the first usable build of the
operator UI. Each item links to the commit on GitHub where the change
landed.

## Memory backend

### Phase 1 — short-term sliding window
The hybrid memory stack starts with a Redis-backed sliding window of the
last N turns, behind a single `MemoryFacade` so chat code never talks to
a raw client.

- Hybrid memory Phase 1 merged
  ([`2508f46`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/2508f46))
- Shared `EmbeddingService` extracted; tool retriever reuses it
  ([`7a4d9fd`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/7a4d9fd))
- Docker compose for the LAN memory stack
  ([`2f84291`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/2f84291))

### Phase 2 — Profile (Postgres) + Episodic (Qdrant)
Long-term identity facts go into a single-row JSONB Profile; per-fact
embeddings + emotion go into Qdrant Episodic. Both layers sit behind the
same facade with 500 ms per-layer circuit breakers.

- System prompt assembles `ContextBundle` per turn
  ([`127406f`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/127406f),
  [`bdce752`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/bdce752),
  [`efea46e4`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/efa46e4))
- Compact-JSON Profile block matches the prune byte cap
  ([`1ddf593`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/1ddf593))
- Debug-only memory REST endpoints (gated by `DEBUG_MEMORY`)
  ([`2def420`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/2def420))
- Health endpoint reports `healthy / degraded / unhealthy`
  ([`c6edcbb`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/c6edcbb))
- Frontend health banner polls `/health`
  ([`35cb371`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/35cb371),

### Phase 3 — LOCOMO worker + canonicalizer

distilled facts to Profile + Episodic. A nightly canonicalizer dedupes
near-duplicate entity tags.

## Terminal control

PTY sessions + one-shot commands run through a Node `shore-pty-service`
microservice. The agent gets a `run_command` tool plus session-aware
read/write tools.

- Terminal feature merged: one-shot + PTY sessions + xterm UI
  ([`23e80eb`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/23e80eb))
- `TerminalService` delegates execution to a `TerminalBackend` interface
  ([`f83d820`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/f83d820))
- `NodePtyClient` (reverse-proxied WS) + config vars
  ([`4f96cea`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/4f96cea))
- node-pty becomes the sole executor; `pywinpty` removed
  ([`7939cfd`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/7939cfd))
- Session tabs, history, and terminal panel in the UI
  ([`f0e3978`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/f0e3978),
  [`2874c0c`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/2874c0c))
- `start_background_service` tool family for detached services
  ([`02a9e3b`](https://github.com/Project-ShoreKeeper/Shore-Assistant/commit/02a9e3b))
- Session-aware terminal shells (pwsh, wsl, anaconda), `read_terminal`,
